## 4. 诚实点 / W2 没解决的

- 🚩 **交叉印证 v0 暴露真问题**：主题字符串跨源不对齐（jobs 写"AI编码agent...必备技能"、blog 写"...当技能"被当成两个单源）。v1 用别名归一临时修好，**但这需要一个真正的 canonical 主题本体 / 模糊匹配——W3+ 必做**，否则规模化后印证会失真。
- 🚩 **单源"待验"里有标注假象**：Python/云/Docker 只从 jobs 打标=1 源，不代表它"弱"，是它们太基础没被前瞻源单列。归一逻辑要能区分"基础技能"与"前瞻信号"。
- 🚩 **多模态信号矛盾**：企业(META 押 AI 眼镜)说热，开源代表仓(LLaVA)却停滞——典型"选错代表仓"。W3 需换多模态代表仓(Qwen-VL 等)。
- 🚩 **博客源覆盖不全**：Anthropic 无干净 RSS、DeepLearning.AI feed 坏、几位大佬近 90 天没更新——前瞻最领先那条腿其实最脆，W3 要加 HF Hub / Hacker News 补冗余。
- 🚩 LLM 抽取仍是我人工做，未自动化 / 未加 schema 约束（W3）。
